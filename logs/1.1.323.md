# Roman Society 1.1.323

Release date: 2026-06-16

This release is a senior-developer pass over the 1.1.322 advanced systems: it fixes
the slave-market money/disappearance bugs, audits the bundled mods, and finishes the
political-rank, spouse-intimacy, and Optimates/Populares features.

## Slavery Fixes (critical)

- Fixed market-bought slaves vanishing on the next monthly turn. Purchased slaves
  were generated with the market template still attached, so they were stored with
  `corSocietySlaveActive: false` and then purged by the monthly `playerSlaveRecords`
  filter. `buySlave` now clears the `market` flag before generating, so a purchase
  always produces an active, owned dependant.
- Fixed money being deducted when a purchase could not actually deliver a slave.
  `buySlave` now secures a valid character (with a real `characterId`) **before**
  charging the buyer, and aborts with no cost if generation/transfer fails.
- Hardened the fallback registration path so distinct slaves are never collapsed:
  dedup is strictly by `characterId`, and records without a `characterId` are dropped
  rather than charged.
- Net effect: buying multiple slaves keeps all of them, the last remaining slave no
  longer disappears, and "buys" that fail no longer drain cash.

## Bundled Mod Audit

- Reviewed every bundled mod (`play_as`, `murder`, `stealingFrom`, `disinheritance`,
  `restoreInheritance`, `bank_of_rome`, `coemptio`, `household_slaves`, `cor_translator`).
  They are thin delegators to the `cor_society` engine; confirmed every delegated
  method (`registerBundledSlaveActions`, `openHouseholdSlaves`, `registerBundledBankActions`,
  `openBankOfRome`, `registerBundledMatchmakerAction`) resolves to a real engine/module
  method, so there are no dead delegations or duplicate slave/bank state paths.

## Political Rank Display

- The Society hub "Date and standing" line now shows the player's political rank
  (`Imperator Augustus` or `Dictator`) when it applies, instead of only the
  property-derived stratum title.

## Spouse Intimacy

- Spouses no longer expose the generic "Court privately" courtship flow (introduction,
  rapport, scandal approaches). Instead they offer a dedicated "Lie together" action
  that reliably strengthens the marital bond and, when the couple can conceive, attempts
  a pregnancy through the shared `impregnate` pregnancy system.

## Optimates and Populares

- Added a real factional layer. Houses are classified as Optimates (senatorial,
  wealthy) or Populares (popular, poorer) via `houseFactionLean`, and the counts are
  written back into `rome.factions`.
- Added a "Optimates and Populares" screen under Roman power where the player can
  champion the Optimates, champion the Populares, or stay unaligned. Declaring a banner
  shifts aligned/opposed house relations.
- Player alignment now biases legislation: the bundled power laws are popularis-flavoured
  (anti-Senate), so championing the Populares raises their pass chance (scaled by the
  popular share of houses) while championing the Optimates lowers it.

## Save Compatibility

- All new reads are guarded for old saves: `playerPolitics.alignment` defaults to
  unaligned, and faction counts recompute from live house data each time they are needed.

## Packaging

- Bumped all module mixin version markers and the engine version from `1.1.322` to `1.1.323`.
- Updated root and runtime README files to `1.1.323`.
- Release artifact: `Releases/CORSociety-v1.1.323.zip`.

## Remaining Validation

- Local object-literal syntax checks validate every module file, but final balance and
  UI flow should still be playtested inside Citizen of Rome because the local workspace
  does not run the full game runtime or DAAPI UI.
