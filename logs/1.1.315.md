# Roman Society 1.1.315

## 1.1.315 - Disable the "Known Family Tree" entry

The vanilla "Known Family Tree" view does not represent Society's houses,
dynasties or generated kin coherently. Instead of patching it further, this
release removes it as a usable entry inside Society. Players use the working
House Family Tree and Dynasty Tree instead.

### Changed

- **Hidden from the Society action list.** `vanillaCharacterActions` now filters
  out any vanilla/other-mod action that opens the known-family screen
  (`isHiddenVanillaFamilyTreeAction`, matched on key/title/tooltip/route/method
  containing "knownfamily" / "known family"). Only the Known tree is hidden; all
  other vanilla actions and the working Society trees are untouched.
- **Entry neutralized.** `openVanillaKnownFamily` no longer opens the vanilla
  known-family route or the Society 'known' mode. It now redirects to the working
  House Family Tree for the character's house (falling back to the character sheet,
  then the hub). So even if some path still triggers it, the broken tree never
  opens.

### Not changed

- The House Family Tree and Dynasty Tree are untouched and keep working.
- No base-game files are modified; the change is entirely inside the mod and is
  reversible (remove the filter call and restore the old `openVanillaKnownFamily`
  body).

### How to test

- Open a character's "Vanilla / other mods actions" in Society: the "Known Family
  Tree" entry should no longer appear.
- Confirm "House tree" and "Dynasty tree" still open normally.

### Notes

- If the base game additionally shows its own native "Known Family Tree" button on
  its own (non-Society) character screen, that is outside the mod's UI; hiding that
  specific base-game button would need its DOM selector. The mod no longer routes
  players to it.
- Runtime version aligned to `1.1.315`.
