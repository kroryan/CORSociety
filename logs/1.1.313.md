# Roman Society 1.1.313

## 1.1.313 - Fix vanilla Known Family Tree duplicating the player (1.1.312 regression)

1.1.312 tried to fix the vanilla tree "split in two" by coupling a child's parents
with `linkParentPair`. That coupling was **too aggressive and not atomic**, so it
created one-directional ("fake") marriages that made the vanilla Known Family Tree
render the same person (e.g. the player) several times.

### Root cause

`linkParentPair` relinked each parent's `spouseId` **independently**. With a child
whose parents were a dead generated ancestor + a living character, the dead
ancestor got `spouseId = livingCharacter`, but the living character was protected
and kept their real spouse. Result: an **asymmetric spouse link** (the dead
ancestor claims the living person; the living person does not claim back). The
vanilla tree, which groups by married couples, then drew the living person in
multiple couples -> the player appeared 2-3 times.

### Fixes

- **`linkParentPair` is now atomic.** It forms the couple only if BOTH sides are
  safe to link; otherwise it changes nothing. No more one-directional marriages.
- **`canRelinkGeneratedSpouse` is stricter.** Only dead ancestors / ghost
  connectors are "owned" and editable. A living character (real OR generated) is
  never given a fabricated spouse and never has an existing marriage overwritten.
  Empty spouse fields are filled only for dead ancestors.
- **New `repairFamilyLinkIntegrity`** (runs bounded on monthly tick and on load):
  repairs existing saves damaged by 1.1.312. For mod-owned characters only, it
  clears self-father/self-mother/self-spouse links, spouse links to missing
  characters, and **non-mutual** spouse links (clearing only our own side, never
  the partner). This removes the duplicate-causing fake links.
- **New `family` debug section** (`corSocietyDebug("family")` or the "Player family"
  button in the debug console): dumps every related character by ID with name, age,
  alive/dead, generated/ghost, resolved house, dynasty, father/mother/spouse,
  children, and flags data issues (self-links, asymmetric spouses, possible
  duplicate name+birthYear with different IDs). Use this to verify by ID.

### Invariants now enforced for mod-owned characters

1. A character is never its own father/mother/spouse.
2. `spouseId` never points at a missing character.
3. A spouse link is only kept if it is mutual (otherwise our side is cleared).
4. A living real marriage is never overwritten, and a living character is never
   given a fabricated spouse.
5. Couples are only formed when both sides are safely linkable (atomic).

### About missing living members

The vanilla Known Family Tree only shows characters connected to the focus by real
parent/child/spouse links. Living house/dynasty members with no recorded biological
link to the player are intentionally not shown there (they still appear in the
mod's House/Dynasty trees, which have a wider scope). This release does NOT
fabricate biological links to force them into the vanilla tree; an unconnected
living relative is a data gap to diagnose (use the `family` debug dump), not a
relationship to invent.

### Changed

- Runtime version aligned to `1.1.313` (invalidates the family-tree cache).
- House and Dynasty trees are unchanged (1.1.311 behaviour preserved).
