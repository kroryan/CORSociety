# Roman Society 1.1.312

## 1.1.312 - Vanilla Known-Family Tree Coherence + "Full" -> "Dynasty" Rename

### Fixed - vanilla KNOWN FAMILY TREE split into two trees

The base game's family tree groups a child's parents as one married couple and
draws the family descending from that couple. The mod generates dead ancestors
to complete a lineage, but a child's two parents were not always linked as each
other's spouse:

- `linkParentPair` only set `spouseId` when the field was **empty**. If a generated
  parent already held a stale or different `spouseId`, the other parent was never
  linked back, so the father and mother of the same child were not a couple.
- `ensureGeneratedParentPair` did not couple parents at all when **both** already
  existed.
- `repairIncompleteGeneratedParentPairs` skipped any character that already had
  both parents, so existing saves were never repaired.

With the parents not mutually married, the vanilla tree rendered the paternal line
and the maternal line as **two separate trees** for the same family.

Root-cause fix (data level, so it benefits the vanilla tree and both mod trees):

- New `canRelinkGeneratedSpouse` decides when it is safe to repair a `spouseId`:
  empty links, stale links pointing at a missing character, dead ancestors, and
  mod-generated ghost connectors are repairable; a **living, non-generated**
  character's valid existing marriage is never overwritten.
- `linkParentPair` now makes the couple **mutual** using that rule (both partners
  point at each other), instead of only filling empty fields.
- `ensureGeneratedParentPair` couples the parents even when both already exist.
- `repairIncompleteGeneratedParentPairs` now couples already-complete parent pairs
  too, so existing saves get repaired over a few ticks (bounded per tick).

Net effect: a child's parents become one couple, so the vanilla Known Family Tree
shows a single connected family instead of two duplicated/parallel trees, and it
no longer contradicts the mod's House/Dynasty trees. Living real marriages are left
untouched; genuinely non-coupled real parents are not force-married.

### Changed - naming

- Every user-facing "Full Dynasty Tree" / "Full dynasty tree" label is now
  "Dynasty Tree" / "Dynasty tree" (the dynasty view title, the dynasty-screen and
  house-screen buttons, and the text-fallback title for the dynasty view).
- The extinct-house button "Full family tree" is now "Family tree" (it shows a
  single house, so it is intentionally not called a dynasty tree).
- Internal tokens are unchanged (`mode: 'full'`, the `#/fullFamilyTree` vanilla
  route, the `fullFamilyTree` DOM id, method names): only visible labels changed.

### Notes / Not Confirmed

- The fix assumes the split is caused by uncoupled parents (the common case for
  generated dead ancestors). If a specific save still shows a split, it may be a
  genuine missing biological link (two ancestors that share no recorded parent);
  that is a data gap, not a coupling bug, and would need targeted ancestor linking.
- Duplicate distinct ancestor *records* for the same logical person were not
  observed in the generators (each guards against regenerating an existing parent),
  but cannot be fully ruled out from static analysis.
- Runtime version aligned to `1.1.312`. Verified statically (all modules parse);
  in-game confirmation of the vanilla tree is recommended.
