# Roman Society 1.1.317

## 1.1.317 - Replace the vanilla "Known Family Tree" portrait button with the House tree

1.1.315 hid the Known Family Tree from Society's own action list, but the base
game's portrait menu (the "view this character's family" button at the top-left of
the portrait) still opened the vanilla Known Family Tree. This release redirects
that entry to Society's working House Family Tree.

### How it works

- `installVanillaFamilyButtonRedirect` patches the game's Vue router (`$router.push`
  and `$router.replace`) once. Any navigation whose target route is `/knownFamily`
  is intercepted and, instead of opening the vanilla tree, Society opens the House
  Family Tree for the currently selected character (`openHouseTreeForVanillaCharacter`).
  Every other navigation passes straight through unchanged.
- The match is on the **route path**, not the button label, so it keeps working in
  any language (including when the CoR Translator has translated the button text).
- It is installed from `boot` (which runs periodically), so it retries until the
  game's router exists and is idempotent (guarded by a version flag on the router).
- If no character is selected or the character has no resolved house, it falls back
  to the Society hub instead of the broken tree.

### Not changed

- House Family Tree and Dynasty Tree are untouched.
- No base-game files are modified; only the router method is wrapped at runtime, and
  the original is preserved (`__corSocietyOriginal_push` / `_replace`) so it is
  reversible.
- The 1.1.315 filter that hides the Known tree from Society's own action list stays
  in place.

### How to test

1. Click the family button at the top-left of a character portrait (the vanilla
   "view this character's family" menu) and choose the family-tree entry.
2. It should now open Society's House Family Tree for that character instead of the
   vanilla Known Family Tree.
3. Confirm normal navigation (other screens/buttons) still works.

### Notes / Not Confirmed

- Assumes the vanilla button routes to `/knownFamily` (the route Society already
  used for that screen). If on your build it uses a different route name, tell me the
  route and I will add it to the matcher.
- Runtime version aligned to `1.1.317`.
